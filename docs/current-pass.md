We are doing Audio Quickfix Pass 7: standardized test matrix.

Goal:
Run the same firmware, same detector params, and same logging format across multiple physical setups to separate software timing issues from acoustic/placement issues.

Do not change detector thresholds between tests.

Test conditions:
- 100 trials each
- same chirp frequency
- same chirp duration
- same volume/output method
- same detector params
- same SEQ window
- same firmware build
- same logging verbosity

Test matrix:
A — current setup / 50 cm
B — 30 cm
C — 50 cm with better mic orientation
D — different cup/body/enclosure

Required output per run:
1. Normalized per-trial lines:
   SEQ trial=<n> result=<expected|early|late|miss|unexpected|duplicate|invalid_audio> dt=<ms|-> dur=<ms|-> strength=<f|0> audio=<ok|overflow>

2. End summary:
   SEQ summary:
   expected=...
   early=...
   late=...
   miss=...
   unexpected=...
   duplicates=...
   invalid_audio=...
   avg_strength=...
   avg_duration=...

3. Audio summary:
   AUDIO summary:
   reads=...
   readBytes=...
   zeroReads=...
   shortReads=...
   maxReadBytes=...
   readErrors=...
   overflow=...
   droppedBlocks=...
   totalSamples=...

4. Signal summary:
   SIGNAL summary:
   blocks=...
   samples=...
   candidates=...
   candidateDrops=...

Add a test label to the run start line:

SEQ start test=A_50cm_current ...
SEQ start test=B_30cm ...
SEQ start test=C_50cm_mic_orientation ...
SEQ start test=D_different_cup ...

Acceptance:
- Each run has a clear test label.
- Each run prints 100 normalized trial lines.
- Each run prints SEQ, AUDIO, and SIGNAL summaries.
- No detector/acoustic params are changed between A–D except the physical setup label.
